# Research Doc Skill · 调研文档技能

> 把一个模糊的「调研 X」请求，变成一篇**结构清晰、证据充分、可直接拿去做决策**的调研文档。
>
> 先调研后下笔 · 一手证据优先 · 摘要先行 · 明确推荐 · 一键发布飞书 / Markdown

---

## 这个仓库是什么

`research-doc-skill` 是一个 Agent Skill（兼容 Codex / Copilot CLI / Claude 等 skill 协议），用来指导 AI agent 完成「调研一个东西并写成文档」这类任务。

它不是一个通用写作 prompt，也不是 PPT 模板。它的核心目标是把**调研这件事本身做对**：

1. **先界定问题**——调研对象是什么、为谁调研、要回答哪 3–5 个关键问题。
2. **去拿一手证据**——读源码 / 实测产品 / 读官方文档 / 搜权威来源，而不是抄二手、凭印象。
3. **结构化呈现**——摘要先行、选项对比、明确推荐、风险与下一步、可追溯来源。
4. **干净发布**——默认建成飞书 docx（标题带前缀、归档知识库、链接 unfurl），也支持纯 Markdown。

一句话：**让 AI 不只是「写一篇看起来像调研的文档」，而是真的把一个问题调研透并给出可执行结论。**

这套方法论是从大量真实调研实践（产品调研、源码仓库调研、功能可行性、能力盘点、现实事务攻略）中沉淀出来的，包含了把文档真正发布、归档、可用过程中踩过的坑。

## 适合谁用

- 经常要写**技术 / 产品 / 源码 / 可行性调研文档**的人。
- 用 AI agent 做内容生产，希望调研类输出**稳定、有结论、有证据**的人。
- 在**飞书 / Lark** 环境里产出文档并归档进知识库的团队（内置全套发布流程与踩坑）。
- 想要一套可复用的「调研 prompt 指令模板」的人。

## 核心方法论速览

### 四阶段工作流

| 阶段 | 做什么 | 关键纪律 |
|---|---|---|
| **0 界定** | 锁定对象 / 读者 / 3–5 个关键问题 | 信息不全就做合理假设写进 Scope，别反复追问 |
| **1 取证** | 按对象类型拿一手证据 | 每个关键结论都要能指回一个来源 |
| **2 综合** | 归纳选项 → 对比 → 下判断 | 给明确推荐，不要罗列完事 |
| **3 下笔** | 套标准骨架 | 摘要最后写、放最前面；能用表/图就别用段落 |

### 四种文档变体

- **产品 / 设计调研**：定位 / 信息架构 / 视觉语言 / 交互 / 差异化，带官方截图与精确数值。
- **源码 / 仓库调研**：基于通读源码的架构剖析 + 核心约束，配 Mermaid 架构图。
- **可行性 / 方案调研**：先卡硬约束，再按推荐度排序给方案。
- **现实事务攻略**：按官方最新规则的时间线组织，附速查清单。

## 适配说明

- 默认输出**飞书 docx**（含 `--as bot` 身份、`[前缀]` 标题、知识库归档、链接 unfurl）。这部分依赖 `lark-cli`。
- 不在飞书环境时，发布章节整段跳过，改用 **Markdown / 本地文件**，调研方法论与文档骨架完全不变。
- 标题前缀、默认知识库等为作者环境约定，换环境时按该环境调整。
