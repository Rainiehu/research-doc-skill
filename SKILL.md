# Research Doc Skill · 调研文档技能

## 核心定位

把一个模糊的「调研 X」请求，变成一篇**结构清晰、证据充分、可直接拿去做决策**的调研文档。

不是写百科全书，不是堆资料，也不是 PPT 信息图。目标是：

1. **先把问题搞清楚**——调研对象是什么、为谁调研、要回答哪几个问题。
2. **去拿一手证据**——读源码 / 读官方文档 / 实测 / 搜权威来源，而不是凭印象写。
3. **结构化呈现**——摘要先行、选项对比、明确推荐、风险与下一步、可追溯来源。
4. **干净发布**——默认建成飞书 docx，标题带前缀、归档进知识库、链接单独发让它 unfurl。

> **⚠️ 第一铁律：不存在的东西绝不编。** 遇到陌生术语 / 缩写（尤其是用户随口给的、可能是 typo 的词），**先核实它真实存在、有出处，再写**；查不到实锤就如实说「查不到」或回头跟用户确认是不是笔误，**绝不替它发明定义、立一节、配一张图**。把「真实的工程概念」如实写，把「杜撰的命名/出处」全部拒绝。详见 `references/research-workflow.md` 取证铁律。

一句话：**让 AI 不只是「写一篇看起来像调研的文档」，而是真的把一个问题调研透并给出可执行结论。**

## 工作流速览

> 详见 `references/research-workflow.md`。

### 1. 先界定调研对象与问题（不要急着搜）

接到「调研 X」时，先在心里（或一句话回给用户）确认三件事：

- **对象**：X 到底是什么？一个产品 / 一份源码仓库 / 一项技术 / 一个方案 / 一件现实事务（如考试、流程）。
- **读者与用途**：调研结果给谁看、要支撑什么决策？决定了深度和侧重。
- **关键问题（3–5 个）**：这篇文档必须回答哪几个问题？没有问题清单的调研一定发散。

不确定时做出**合理假设并写进文档的「范围」一节**，不要停下来反复追问。

### 2. 去拿一手证据（取证优先于下笔）

按对象类型选取证手段，**能拿一手就不要二手**：

- **源码 / 仓库调研**：通读关键路径源码，提炼真实约束（例：「adapter 都是一次性子进程、stdin 即关」这种结论必须来自读码，不能猜）。
- **产品 / 设计调研**：访问官网 / 实际产品、抓官方截图、读官方博客与发布说明；网传二手说法要用一手来源校正。
- **技术 / 方案调研**：读官方文档、RFC、论文、benchmark；动手能跑就跑一下。
- **可行性调研**：先确认「官方到底开不开放」这类硬约束，再谈方案。
- **现实事务攻略**：以官方规则为准（费用、流程、评分标准），不要用过时经验。

每一个关键结论都要能指回一个来源。

### 3. 综合：从证据到结论

- 把发现归纳成**几个选项 / 几条路径**，做**对比表**。
- 给**明确推荐**（哪个、为什么），而不是罗列完事让读者自己选。
- 写清**风险与缓解**、**下一步**。

### 4. 下笔：套标准骨架

用 `references/doc-structure.md` 的骨架。**摘要最后写、但放最前面**。复杂结构用对比表、用画板（结构 / 对比 / 光谱 / 评分一律配图，见 `references/whiteboards.md`）、用配图。**画板是默认动作不是可选**：每篇至少 1 张，复杂对象 3–5 张。**配图先选对工具——表达第一、美观第二**：真实流程/循环/因果用 Mermaid，矩阵/卡片/光谱/计分用 SVG（见 `whiteboards.md`「先选对工具」）。

> **迭代加深 vs 定期重构**：用户常对已发布文档连续追问，每次追加一节；尾部补丁攒到 2–3 个就**主动整篇重排成逻辑主线**，别让文档长成流水账（见 `doc-structure.md`「定期重构」）。

### 5. 发布与归档

默认飞书 docx：`--as bot` 身份建 → 标题加 `[YuCopilot] ` 前缀并**显式 update_title** → `wiki +move` 进知识库（默认 AI Related）→ 把链接**单独发一条 text 消息**让它 unfurl。细节与所有踩坑见 `references/feishu-publishing.md`。

### 6. 发布前过一遍质量清单

见 `references/quality-checklist.md`。重点：每个关键结论有来源吗？有没有明确推荐？对比表是不是真的在对比？标题对吗？

## 默认与可调项

- **默认输出**：飞书 docx（适配本环境）。用户要 Markdown / 本地文件时切换，骨架不变。
- **默认身份**：`--as bot`（机器人自己），标题前缀 `[YuCopilot] `。这是当前用户的约定，换环境时按该环境约定调整。
- **默认归档**：飞书知识库「AI Related」。
- **语言**：跟随用户，默认中文正文 + 必要的英文术语。
