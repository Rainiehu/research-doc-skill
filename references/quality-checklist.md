# 发布前质量清单

调研文档最危险的失败是「看起来完整，其实空洞」——结构齐全但没有结论、没有证据。发布前逐条过这份清单。

## 内容（最重要）

- [ ] **没有编造的概念 / 术语 / 出处吗？**（第一铁律）文档里每个专有名词、缩写、范式名，都是真实存在、能 `web_search` 到出处的吗？有没有把用户的口误 / typo（如「grain」）当成既有理论去造定义、立节、配图？查不到实锤的，要么删掉、要么标「未证实」、要么回头跟用户确认——**绝不发明**。
- [ ] **有没有明确结论？** 摘要第一句就能让读者拿到核心结论和推荐，而不是「以上是一些情况」。
- [ ] **每个关键结论有来源吗？** 随手指一个判断，能说出它来自哪段源码 / 哪个官方页面 / 哪条数据吗？凭印象写的句子删掉或去补证。
- [ ] **关键问题都回答了吗？** 对照「背景」里列的 3–5 个关键问题，逐个确认正文有答。
- [ ] **有推荐而不只是罗列吗？** 多选项时给了「选哪个、为什么、放弃别的原因」，而不是把球踢给读者。
- [ ] **硬约束确认过吗？**（可行性类）推荐的方案没有建立在「其实官方不开放 / 不合规」的错误前提上。
- [ ] **数字有出处和适用范围吗？**（攻略类）费用 / 时限 / 评分标注了来源与地区/年份。
- [ ] **一手 vs 二手分清了吗？** 网传说法有没有用一手来源校正过。

## 结构与呈现

- [ ] 摘要在最前面，且是最后写的（反映全文结论）。
- [ ] 选项 / 特性对比用了**表**，而不是大段文字。
- [ ] 结构性内容（架构 / 流程 / 分类 / 时间线 / 对比 / 光谱 / 评分）用了**画板**，并已插进文档对应小节、核验过线上渲染。
- [ ] **画板数量够吗？** 每篇至少 1 张，复杂对象 3–5 张（见 `references/whiteboards.md`）。
- [ ] 产品 / 界面类有**截图或渲染图**。
- [ ] 每节自包含，标题信息量足。
- [ ] 有「来源 / 参考」节。

## 发布（飞书）

- [ ] 标题正确，带 `[YuCopilot] ` 前缀，且**已用 `update_title` 显式设置并核验**（不是 Untitled）。
- [ ] 用 `--as bot` 身份创建。
- [ ] 已 `wiki +move` 进知识库（默认 AI Related）。
- [ ] 文档链接**单独发了一条 `msg_type:text` 裸链接**让它 unfurl。
- [ ] 临时文件已清理。

## 交付给用户

- [ ] 先说一句话核心结论，再给结构概览和链接。
- [ ] 没有照抄 bridge 注入的元数据（`<bridge_context>` 等）到回复里。
